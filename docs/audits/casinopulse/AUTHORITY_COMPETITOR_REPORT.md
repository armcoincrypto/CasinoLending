## Competitor Profiles

### 1. CasinoGuru (casino.guru)

**Estimated referring domains:** ~21,000 (main domain; locale properties vary)

**Top linked page types:**
- Casino review pages with fairness/complaint data
- “Best casinos in [country]” programmatic hubs
- Responsible gambling + license verification tools
- Complaint submission/resolution flows

**Content types attracting links:**
- Original testing methodology (simulated play, bonus T&C audits)
- Player complaint database (citable trust signal)
- Multilingual subdomain/domain architecture (hreflang at scale)
- Annual/industry transparency reports

**Authority sources:**
- Gambling forums referencing complaint outcomes
- Affiliate/industry blogs citing fairness scores
- Regional gambling communities linking “sites to avoid”
- Wikipedia-adjacent citations on licensing topics (indirect)

**Common anchor patterns:**
- Brand anchors: “Casino Guru”, “casino.guru”
- Descriptive: “casino reviews”, “online casino complaints”, “fairness testing”
- Naked URLs on forum threads

**CasinoPulse gap:** No complaint product, no searchable casino database page, no methodology landing page with citable framework.

## Strategic Implication

CasinoPulse cannot close the referring-domain gap with volume link building. The viable path mirrors incumbents:

1. **Ship citable assets** (reports, databases, methodology)
2. **Digital PR** around original South Asia + crypto payment data
3. **Entity building** (consistent profiles, expert visibility)
4. **Topical cluster expansion** with depth, not thin doorway pages
5. **Internal authority flow** to money/intent pages already built

**12-month realistic target:** 200–500 quality referring domains (editorial + regional + crypto), not 10,000.
